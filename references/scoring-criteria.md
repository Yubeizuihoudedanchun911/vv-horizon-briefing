# 评分标准

每条内容按 5 个维度独立打分，每维度 0-2 分，总分 0-10。

## 评分维度

| 维度 | 0 分 | 1 分 | 2 分 |
|------|------|------|------|
| 新颖性 | 已知信息，无新角度 | 已知话题的新视角、新数据或增量改进 | 全新概念、首次公开发布或突破性成果 |
| 影响力 | 小众或局部影响 | 对特定社区有实用价值（如某语言生态、某工具用户群） | 跨生态系统影响或潜在主流采用（>100 万潜在用户） |
| 技术深度 | 表面介绍，无实质内容 | 有实质技术内容（含代码片段、架构说明或可操作步骤） | 深度技术，含完整实现、论文或可独立复现的方法 |
| 社区信号 | 低于该源基准值 | 达到该源基准值 1-3 倍 | 超过该源基准值 3 倍以上 |
| 时效性 | 发布超过 48 小时 | 发布 24-48 小时内 | 发布 24 小时内 |

## 各源社区信号基准值

| 数据源 | 基准值 |
|--------|--------|
| HackerNews | score ≥ 100 |
| Reddit | upvotes ≥ 50 |
| GitHub releases | stars ≥ 50 或 release notes 非空 |
| RSS / 微信公众号 | 无互动数据时默认得 1 分 |

## 评分规则

- 各维度独立打分，求和得总分（0-10）
- **社区信号**参考各源基准值（见上表），而非跨源比较绝对数字
- 无正文内容（仅标题）的条目，技术深度最高得 1 分
- 评分后按总分降序排列，取 `config_snapshot.top_n` 条进入后续分析
- 同分时，优先取时效性更高的条目

## 示例

### 高分示例（10/10）

HackerNews 一篇关于新 LLM 架构的文章，score=450，发布于 6 小时前：
- 新颖性: 2（全新架构，首次公开）
- 影响力: 2（影响整个 LLM 领域，潜在用户数百万）
- 技术深度: 2（含论文和完整代码）
- 社区信号: 2（HN 基准 100，450 超过 3 倍）
- 时效性: 2（6 小时内）
- **总分: 10/10**

### 中等示例（5/10）

Reddit r/LocalLLaMA 一篇关于量化技巧的帖子，upvotes=60，发布于 30 小时前：
- 新颖性: 1（已知量化话题，但有新的对比数据）
- 影响力: 1（对本地 LLM 用户有实用价值，但受众有限）
- 技术深度: 1（有代码片段，但无完整实现）
- 社区信号: 1（Reddit 基准 50，60 在 1-3 倍范围内）
- 时效性: 1（30 小时，在 24-48 小时范围内）
- **总分: 5/10**
